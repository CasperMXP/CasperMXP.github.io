jdk7后intern()方法

  • intern方法还是会先去查询常量池中是否有已经存在，如果存在，则返回常量池中的引用
  • 原来在常量池中找不到时，复制一个副本放到常量池
  • 1.7后则是将在堆上的地址引用复制到常量池。

示例1

  • 当执行String str1 = new String(“abc”)的内存如下，常量池有一个abc,以及堆里面有abc

  • 当执行str1.intern()的时候发现常量池已经有了，直接返回常量池
  • 所以此时str1==intern为false

示例2

  • 执行String str1 = a + b;的时候利用的StringBuilder的append()方法并没有生成常量
  • 执行str1.intern()的时候，生成的常量将是引用指向str1,所有intern==str1为true
