类的加载，连接和初始化

JVM进程将被终止

  • 程序运行到最后正常结束
  • 程序运行到使用System.exit()或Runtime.getRuntime().exit()代码处结束程序
  • 程序执行过程中遇到未捕获的异常和错误而结束
  • 程序所在平台终止了JVM进程

类的加载

类的连接

  • 验证：验证阶段用于检验被加载的类是否有正确的内部结构，并和其他类协调一致。
  • 准备：类准备阶段则负责为类的类变量分配内存，并设定默认值。
  • 解析：将类的二进制数据的中的符号引用替换成直接引用。

类的初始化

主要是对类变量做初始化，JVM初始化一个类主要包含以下几个步骤：

  • 假如这个类还没有被加载和连接，则程序先加载并连接其类
  • 假如该类的直接父类还没有被初始化，则先初始化其直接父类
  • 假如类中有初始化语句，则系统依次执行这些初始化语句

类的初始化时机：

  • 创建类的实例，new,反射，反序列化
  • 调用某个类的类方法
  • 调用某个类或接口的类变量或为该变量赋值
  • 使用反射方式强制创建某个类的或接口对应的java.lang.Class对象
  • 初始化某个类的子类，所有父类都会被初始化
  • 直接使用java.exe命令来运行某个主类

类加载器

类加载器负责将.class文件加载到内存中，并生成对应的java.lang.Class对象。

根类加载器 Bootstrap ClassLoader

  • 负责加载Java的核心类

扩展加载器 Extention ClassLoader，

  • 负责加载JRE扩展目录JAVA_HOME/jre/lib/ext或java.ext.dirs目录指定的jar包

系统类加载器 System ClassLoader

  • 加载java命令的classpath选项
  • java.class.path系统属性
  • CLASSPATH环境变量所指定的jar包和类路径

类的加载机制

全盘负责

当一个类加载器负责加载某个Class的时候，该class所依赖的和引用的其他Class也将由该类加载器负责载入

父类委托

先让父类加载器试图加载该class,只有在父类加载器无法加载该类的时候才能尝试从自己的类路径加载该类

缓存机制

缓存机制将保证所有被加载过的class都会被缓存，当程序需要使用某个class对象的时候，类加载器先从缓存中搜寻该class。

创建并使用自定义的类加载器

ClassLoader类有2个关键方法

  • loadClass(String name,boolean resolve):该方法为ClassLoader的入口点，根据指定名称来加载类
  • findClass(String name)根据指定名称来查找类

URLClassLoader

是系统加载类和扩展类的父类，可以从本地文件系统获取二进制文件来加载类，可以从远程主机获取二进制文件来加载类。
