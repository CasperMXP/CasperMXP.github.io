单元测试概述

按照软件工程的思想，软件测试可以分为单元测试、集成测试、功能测试、和系统测试等。功能测试和系统测试一般来说是测试人员的职责，但是单元测试和集成测试则必须是开发人员保证的。单元测试就是开发者编写的一段小代码。通常而言，一个单元测试用于判断某个特定条件后特定场景下某个特定函数的行为。

单元测试系统基本概念

  • 被测系统SUT:System Under Test或AUT Application Under Test
  • 测试替身Test Double：使用测试替身减少被测对象的依赖，使得测试更加单一。
  • 测试夹具Test Fixture：测试运行程序，会在测试方法运行之前自动执行初始化，回收资源
  • 测试用例 Test Case
  • 测试套件 Test Suite将多个测试用例组成一个测试用例组，则测试套件批量执行。

参数化测试

为了测试程序的健壮性，可能需要模拟不同的参数对方法进行测试，如果为每个类型的参数创建一个测试方法，则是意见很难过事情。幸好的是TestNG提供了参数化测试，它能够创建由参数值供给的通用测试，从而为每个参数都运行一次，而不必要创建多个测试方法。

套件测试

测试套件是用于测试软件程序的行为或一组行为的测试用例的集合。在TestNG中无法再源代码中定义一个套件，但是它可以由一个XML文件表示，因为套件是执行的功能，套件可以包含一个或多个测试，并由标记定义。是testng.xml根标记

其中suite标签有以下属性
|属性|描述|
|:—–:|:—–:|
|name|套件的名称，必需的属性|
|verbose|运行的级别或详细程度|
|parallel|TestNG是否运行不同的线程来运行这个套件|
|thread-count|如果启用并行模式，则要使用的线程数|
|annotations|在测试中使用的注释类型|
|time-out|在本测试中所有测试方法上使用默认的超时时间|

测试整合之王Unitils

Unitils的功能特点：

  • 自动维护和强制关闭单元测试数据库
  • 简化单元测试数据库连接的设置
  • 简化利用DbUnit测试数据的插入
  • 简化HibernateSession的管理
  • 自动测试与数据库相映射的Hibernate映射对象
  • 易于把Spring管理的Bean注入单元测试中，支持在单元测试中使用Spring容器的Hibernate SessionFactory
  • 简化EasyMock，Mock对象创建
  • 简化Mock对象注入，利用反射匹配EasyMock匹配参数
