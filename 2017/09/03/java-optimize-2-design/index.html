善用设计模式

单例模式

  • 对于频繁使用的对象，可以省略创建对象花费的时间
  • 由于new操作的次数减少，对系统内存的使用频率也会降低，减轻GC压力，缩短GC停顿时间
  • 单例模式使用内部类来维护单例模式的实例，当Singleton被加载的时候，其内部类并不会初始化，实现了延迟加载的作用；
  • 当调用getInstance方法的时候才会初始化singleton，由于实例的建立是在类加载时完成的，天生对多线程友好。

代理模式

  • 将代理模式实现延迟加载，可以有效的提高系统的启动速度，对改善用户体验有很大的帮助。hibernate的延迟加载就是利用cglib代理的实现延迟加载。

装饰者模式

  • 可以动态添加对象的功能
  • 装饰者模式可以有效的分离性能组件和功能组件，从而提升模块的可维护性并增加模块的复用性
  • 装饰者和被装饰者具有相同的接口。
  • 被装饰者通常是系统的核心组件，完成特定的功能目标。
  • 装饰者可以在被装饰者的方法前后，加上特定的前置处理和后置处理，增强被装饰者的功能。

JDK中IO就是常见装饰者模式

  • OutputStream公共接口
  • FileOutputStream是具体的被装饰者
  • FilterOutputStream装饰者抽象类对象，并持有一个具体的被装饰者对象
  • BufferedOutoutStream 具体的装饰者对象

观察者模式

  • 观察者模式可用于时间监听，通知发布等场合。可以确保观察者在不使用轮询的监控情况下，及时收到相关的消息和事件。
  • 主题接口，当其状态发生改变或某事件发生时，它会将这个变化通知观察者，它维护了观察者所需要的依赖的状态。
  • 具体主题，具体的主题的实现者，

命令模式

  • Command统一的命令接口
  • ConcreteCommand，具体的命令实现者
  • Receiver命令的实际执行者
  • Invoker命令的请求者，是命令模式中最重要的角色，对各个命令进行控制

常用的优化组件和方法

缓冲Buffer

  • 是一块的特定的内存区域，开辟缓冲区的目的是缓解应用程序上下层之间的性能差异，提高系统的性能。

缓存

  • 提高系统性能而开辟的内存空间，缓存的主要作用是暂存数据处理结果，并提供给下一次的调用。

对象复用-池

  • 数据库连接池
  • 线程池
