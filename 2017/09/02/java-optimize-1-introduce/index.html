性能的概述

看懂程序的性能

  • 执行速度：程序的反映是否迅速，响应时间是否足够短
  • 内存分配：内存分配是否合理，是否过多的消耗内存或者存在内存泄漏
  • 启动时间：程序从运行到可以正常处理业务需要花费多长时间
  • 负载承受能力：当系统压力上升时，系统的执行速度，响应时间的上升曲线是否平缓

性能的参考指标

  • 执行时间：一段代码从开始运行到结束运行，所使用的时间
  • CPU时间：函数或者线程占用CPU的时间
  • 内存分配：程序在运行的时候所占用的内存空间
  • 磁盘吞吐量：描述I/O的使用情况
  • 网络吞吐量：描述网络的使用情况
  • 响应时间：系统对某用户行文或事件作出的响应的时间。

系统瓶颈的计算资源

  • 磁盘IO,读写速度要比内存慢很多
  • 网络操作，网络操作可能比磁盘IO读写更慢
  • CPU,对于长时间，不间断的大量占用CPU资源将导致CPU争夺导致性能下降
  • 异常，异常的捕获和处理是非常消耗资源的
  • 数据库，海量数据的读写操作可能相当费时间
  • 锁竞争，会导致明显的上下文切换的开销
  • 内存，高频率的内存交换和扫描，内存大小不足

性能优化的层次

设计的优化

处于调优手段的上层，它往往需要在软件开发之前进行。进行设计优化的时，设计人员必须熟悉常用的软件设计方法，设计模式，基本性能组件和常用优化思想，并将其有机的集成在软件系统中。

代码调优

正确的使用相关的API或类库，同时对算法或数据结构灵活运用。

JVM调优

堆内存结构，GC种类

数据库调优

  • SQL语句优化
  • 数据库进行优化
  • 数据库软件进行优化

操作系统调优

虚拟内存大小，磁盘的块大小等参数
